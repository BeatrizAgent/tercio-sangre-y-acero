# Tercio Web

Next.js client for **Tercio: Sangre y Acero**.

## Data Flow

- Root `data/*.json` is the source of truth.
- `web/scripts/sync-data.mjs` copies JSON into `web/data/json`.
- `web/src/lib/game-data.ts` maps JSON into runtime definitions.
- `data/assets.json` links gameplay records to `GPT-ASSETS`.

## Commands

```bash
pnpm install
pnpm prisma:generate
pnpm dev
pnpm validate
pnpm build
```

## Assets

New images come from manual ChatGPT prompting, are stored in `GPT-ASSETS`, cleaned by `scripts/process_gpt_assets.py`, and indexed by `scripts/build_asset_bank.py`.

Harsh historical events must stay SFW. Use `mature: true` and `presentation: "blurred"` for indirect, obscured visuals.

Runtime assets are read from `web/public/assets/gpt-bank`, which is generated by `web/scripts/sync-data.mjs`. Do not edit that mirror by hand.

- Missions and mission cards use full CG scenes from `GPT-ASSETS/CG/cg_events`.
- UI chrome uses `icons-ui` through `uiIconPaths` in `web/src/lib/game-data.ts`.
- Diego uses `prota`, `prota/emociones`, and `prota/sprites-animation`.
- Inventory and shop images use `assetId` links from root JSON data.

The full resource map lives in the root `README.md`.
