# Asset Gap Report

This file is a human-friendly mirror of
[`data/_audit/asset_gap_report.json`](../data/_audit/asset_gap_report.json).
It is regenerated automatically by
[`scripts/audit_asset_links.py`](../scripts/audit_asset_links.py) and
is the source of truth for "what is still missing" in the GPT
asset bank.

The most recent regeneration lives in
[DESIGN/recursos-faltantes.next.md](../DESIGN/recursos-faltantes.next.md)
as a per-batch todo.

## How to read it

- **Categorias por planear** lists the eleven batches that the
  agent prepares. For each batch you see how many PNGs the
  category already has, how many are planned, and the prompt file
  to use for the next ChatGPT run.
- **Vinculos faltantes** lists data records that should carry an
  asset link but do not, or that point to a non-existent asset.
  These are the records the user-visible UI will fall back from
  until the bank is filled.
- **Referencias rotas** lists asset references that point to a
  deleted asset. They are never allowed to ship; the validator
  fails the build on any of these.

## Current state

The numbers in this document are stale by definition; the
authoritative version is regenerated by:

```bash
python scripts/audit_asset_links.py
```

To force-fail on any gap use the strict mode:

```bash
python scripts/audit_asset_links.py --strict
```

This is what the CI should run once the bank is large enough that
no gap is acceptable.
